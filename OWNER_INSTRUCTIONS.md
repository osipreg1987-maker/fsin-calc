## 🛡️ 1. О проекте (Суть и Ценность)
**FSIN Calc Pro** (`fsin-calculator.ru`) — это профессиональный веб-сервис и Telegram-экосистема для сотрудников и пенсионеров Уголовно-исполнительной системы (ФСИН).
* **Основная задача:** Автоматизировать сложнейший ручной расчет денежной компенсации взамен неполученного вещевого имущества (обмундирования).
* **Сложность расчета:** Логика учитывает пол сотрудника, специальное звание (Генералы, Полковники, Офицеры, Младший начсостав), нормы положенности (по Приказу ФСИН №211) и рассчитывает износ/недонос пропорционально дням и месяцам службы в конкретных периодах.

### Стек технологий:
* **Frontend/Backend:** Next.js (App Router, React, TypeScript).
* **Стилизация:** TailwindCSS + Framer Motion (для плавной премиальной анимации и эффектов свечения в темной теме).
* **База данных и Авторизация:** Supabase (Auth + PostgreSQL + Realtime-чат поддержки).
* **Интеграция с мессенджером:** Telegram Bot API (через фреймворк Telegraf на бэкенд-роутах Next.js).

## ⚙️ 2. Архитектура и Логика компонентов

### 2.1 Калькулятор (`web/src/components/Calculator.tsx`)
Это ядро приложения.
* **Логика блокировки (`isLocked`):**
  * Чтобы пользователи не пересчитывали данные бесконечно на одном аккаунте (обходя тарифы), после клика на **«Выполнить расчет»** интерфейс замораживается (`isLocked === true`).
  * При `isLocked === true` блокируются: ФИО, звание, основание увольнения, кнопки выбора пола, периоды службы и складской журнал. Скрывается кнопка «Выполнить расчет» и появляется предупреждающая плашка с замочком 🔒.
  * Для нового расчета добавлена кнопка **«➕ Новый расчет»**, которая сбрасывает все состояния в исходные и разблокирует поля.
* **Облачный архив расчетов:**
  * Интегрирован прямо в левый сайдбар калькулятора.
  * Для гостей выводится блок с предложением войти в систему.
  * Для авторизованных пользователей выводится вертикальный список карточек их сохраненных расчетов.
  * При клике на карточку из архива данные загружаются в калькулятор в **100% read-only режиме** (пользователь может скачивать Excel/Word отчеты, просматривать результаты, но не может изменить ни единого поля).
* **Реферальная система:**
  * В калькуляторе есть блок реферальной программы. Метод `getReferralLink` формирует ссылку: `https://www.fsin-calculator.ru/?ref=КОД_ПОЛЬЗОВАТЕЛЯ`. За каждого приглашенного друга, который зарегистрируется, пользователь получает +1 месяц PRO-доступа бесплатно.

### 2.2 Арматурная карточка (`web/src/components/IssueLogTable.tsx`)
* Таблица фактически полученного имущества на складе.
* Передает данные о количестве и датах выдачи в ядро расчета.
* При `isLocked === true` полностью отключает интерактивные селекты выбора вещей, инпуты количества/дат и скрывает кнопки удаления/добавления строк.

### 2.3 Таблица результатов (`web/src/components/ResultsTable.tsx`)
* Красивые дашборды («Положено компенсации», «Подлежит удержанию» и «Итог к выплате») с эффектами свечения в темной теме.
* Содержит детальный пропорциональный расчет по каждой позиции вещевого имущества.
* Скрывается, если расчет еще не произведен (`isLocked === false`), заменяясь на стильную заглушку-подсказку.

### 2.4 Telegram-бот (`web/src/app/api/telegram/bot/route.ts`)
Бот полностью интегрирован в Next.js как API-роут на Webhook'ах.
* **Функционал:**
  * Открытие калькулятора прямо внутри Telegram как встроенное Web App приложение.
  * Скачивание шаблона рапорта в формате `.docx` напрямую в чат.
  * Выгрузка готовых Excel-справок расчетов (на выплату, на удержание, обоснование) прямо в чат для PRO-пользователей по команде `/my_archive` или по кнопке.
  * Двусторонний чат поддержки: сообщения, отправленные пользователем в личку боту, пересылаются в специальную закрытую админ-группу поддержки в Telegram, а ответы админов пересылаются пользователю обратно в личку бота (а также дублируются на сайт через Supabase Realtime).

## 🛠️ 3. Что мы настроили и изменили (Хронология)

1. **Маркетинг на Лендинге (`page.tsx`):**
   * Разместили контрастные бейджи **«ГАРАНТИЯ 5 РАСЧЕТОВ»** и **«ГАРАНТИЯ 30 РАСЧЕТОВ»** в тарифной сетке.
   * Написали крупные условия гибридной подписки (PRO-доступ не сгорает по времени, если не лимитировано число сохраненных уникальных расчетов).
2. **Параметр Paywall в URL:**
   * Реализовали отслеживание query-параметра `?buy_pro=true` при входе в калькулятор. При его наличии автоматически открывается окно оплаты подписки, а сам параметр бесследно удаляется из адресной строки браузера через `window.history.replaceState` без перезагрузки страницы.
3. **Логика фиксации расчетов:**
   * Внедрили стейт `isLocked` и протянули его во все дочерние инпуты, селекты, журналы выдачи предметов и периоды службы.
4. **Сайдбар-архив:**
   * Полностью удалили старый нижний плавающий бар и модальное окно архива.
   * Перенесли весь архив в красивую карточку внутри левой боковой панели с кнопкой быстрого сброса «➕ Новый расчет».
5. **Подключение собственного домена:**
   * Домен **`fsin-calculator.ru`** (и `www.fsin-calculator.ru`) привязаны к хостингу Vercel. На DNS-серверах прописана `A`-запись на IP Vercel и `CNAME` для `www` на Anycast-домен Vercel.
6. **Исправление перенаправлений бота на собственный домен:**
   * Изменили жестко прописанные бэкап-ссылки в бэкенд-коде бота (`src/app/api/telegram/bot/route.ts`). Заменили технический домен `fsin-calc.vercel.app` на новый официальный домен `https://www.fsin-calculator.ru`.
   * Теперь при клике на клавиатуру бота «Открыть калькулятор» или команду `/calc` Telegram открывает Web App строго на новом домене.
7. **Стабилизация реферальных ссылок:**
   * Изменили функцию `getReferralLink` в калькуляторе. Теперь она генерирует ссылки исключительно на домен `https://www.fsin-calculator.ru`, независимо от того, с какого адреса зашел сам пользователь.
8. **Кастомизация хедера:**
   * Сохранили премиальный оригинальный дизайн хедера в самом калькуляторе (крупный герб ФСИН справа, эффектный неоновый бэкграунд-градиент, широкие отступы) и убрали нерелевантный блок нормативной базы с битой ссылкой на стороннюю инструкцию.

## 🔑 4. Критические внешние настройки (Чек-лист для восстановления)

Если проект переносится на новый Vercel или Supabase, обязательно настройте следующие параметры:

### 4.1 Переменные окружения (Environment Variables) в Vercel:
* `NEXT_PUBLIC_SUPABASE_URL` — ссылка на ваш проект Supabase.
* `NEXT_PUBLIC_SUPABASE_ANON_KEY` — анонимный публичный ключ Supabase.
* `SUPABASE_SERVICE_ROLE_KEY` — **секретный сервисный ключ** (крайне важен для обхода RLS прав при связывании Telegram ID и записи данных).
* `TELEGRAM_BOT_TOKEN` — токен вашего Telegram-бота, полученный от `@BotFather`.
* `NEXT_PUBLIC_SITE_URL` — должен быть равен `https://www.fsin-calculator.ru` (используется ботом как базовый адрес сайта).

### 4.2 Настройки в панели Supabase:
1. **Authentication -> URL Configuration:**
   * **Site URL:** `https://www.fsin-calculator.ru` (это решает проблему возврата на старый домен версела при логине!).
   * **Redirect URLs:** Должен содержать `https://www.fsin-calculator.ru/**`.
2. **База данных (Таблицы):**
   * `subscriptions` — профили пользователей, статус PRO, лимиты, реферальный код, привязанный `telegram_id` и `telegram_link_token`.
   * `archives` — сохраненные расчеты (связаны по `user_id` с таблицей пользователей).
   * `support_messages` — переписка поддержки.
3. **RPC Функции (Database Functions):**
   * `log_pro_calculation` — функция списания лимита расчетов у PRO-пользователей.
   * `link_referral_code` — функция связывания рефералов.

### 4.3 Настройки в @BotFather:
* Webhook бота регистрируется переходом в браузере по адресу:
  `https://www.fsin-calculator.ru/api/telegram/bot` (ответом должно быть `{ "ok": true, "message": "Webhook successfully set..." }`).
* Кнопка меню (Menu Button) настраивается в BotFather и должна вести на:
  `https://www.fsin-calculator.ru/calc`
